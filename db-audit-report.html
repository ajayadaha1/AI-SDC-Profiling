Key Finding: The profiler queries only 4 of 26 available tables

The profiler currently uses MSFT_MCEFAIL, LEVEL3DEBUG_LOGFILES, AURA_PMDATA, and PRISM_PMDATA. This misses ~1.9 billion rows of potentially valuable data including production MCE logs, command pass/fail history, part lifecycle data, and SDC detection events. Additionally, ACF (Accuracy Check Failure) rows in LEVEL3DEBUG use ACF_FAIL_CORE_IDS arrays instead of MCA_BANK, requiring different query logic.
